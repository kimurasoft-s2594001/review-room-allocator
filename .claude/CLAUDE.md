# Claude 协作规范（核心版）

> 💡 详细规范请查看根目录 `CLAUDE.md`

## 语言设定

- 中文交流和注释
- 代码命名遵循项目约定

## Git 操作必须询问 ⚠️

执行前**必须询问**：

- ✋ git add / commit / push
- ✋ git merge / rebase / reset
- ✋ 创建新分支

询问时说明：操作内容、涉及文件、主要变更

### 提交信息格式

```
<类型>: <简短描述>

类型：feat, fix, docs, style, refactor, perf, test, chore

❌ 禁止添加 Claude Code 标记
✅ git commit -m "feat: 添加用户登录功能"
```

## 代码操作规范

**重大操作需要确认方针一致**：

- 🗑️ 删除代码/文件/目录 - 首次确认方针
- ⚠️ 修改公共 API/函数签名 - 确认影响范围
- 🏗️ 修改架构/技术栈/引入依赖 - 确认技术选型
- ⚙️ 修改配置文件/环境变量 - 确认配置策略
- 📦 批量操作/全局替换 - 确认替换规则

**方针一致后可直接执行**：

- ✅ 用户已明确方针和目标
- ✅ 继续执行已确认的任务
- ✅ 相同类型的后续操作

## 修改原则

```
✅ 最小化 - 只改必要部分，一次一件事
✅ 向后兼容 - 保持 API 稳定，废弃而非删除
✅ 增量开发 - 小步提交，保持可工作状态
```

## 核心要求

1. **命名**：见名知意，保持一致
2. **注释**：说明为什么，不重复代码
3. **风格**：遵循项目现有规范
4. **安全**：验证输入，保护敏感信息
5. **沟通**：提供上下文，明确需求
